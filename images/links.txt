Q2: Securely execute user Python code without getting hacked
This is a great technical question — here's a solid answer:

I would never run user code directly on the main server. Instead, I'd use one of these approaches:
First option — use an existing sandboxed execution API like Judge0 or Piston API, which are built exactly for this. They handle isolation, timeouts, and resource limits securely.
Second option — if building in-house, I'd spin up an isolated Docker container per execution request with no internet access, read-only filesystem, strict CPU and memory limits (e.g. 256MB RAM, 5 second timeout), and a non-root user. After execution the container is destroyed immediately.
Either way I'd also sanitize the code before sending, block dangerous imports like os, subprocess, and socket, and log all executions for monitoring.
